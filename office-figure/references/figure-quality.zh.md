# 高质量科研图规则

## 1. 先定义图的任务

动手前先用文字确定：

- 读者看完这张图应理解什么。
- 图属于什么领域，应使用哪些真实术语。
- 目标载体是 Word、PPT、LaTeX 还是独立图片。
- 图需要强调结构、数据、流程、对比、机制还是视觉概念。
- 哪些内容必须可编辑：标签、箭头、模块、数据、公式、图例、标题。

不要直接从代码开始。先写一版简短画图指令，包含布局、模块、连线、视觉重点和交付格式。

## 2. 格式选择

| 场景                                            | 优先格式                            |
| ----------------------------------------------- | ----------------------------------- |
| LaTeX 论文、数学公式、精确结构、PGFPlots 数据图 | TikZ / PGFPlots                     |
| Word/PPT 中可继续编辑的框图、路线图、架构图     | draw.io                             |
| PPT 中需要强视觉冲击、背景、纹理、概念插画      | 生成图片 + PPT/draw.io 可编辑覆盖层 |
| Word 中需要正式插图                             | draw.io/SVG 或 TikZ/PDF，必要时 PNG |
| 量化数据图                                      | PGFPlots、可复现脚本或 PPT 原生图表 |

简单结构图不需要过度装饰。毕业论文、学位论文、黑白打印友好图优先白底、黑灰线、少量低饱和强调色。

## 3. 生成图片的边界

生成图片适合做：

- 背景氛围
- 纹理
- 概念插画
- 图标素材
- 风格参考层
- 视觉摘要中的非精确信息层

生成图片不适合承载：

- 精确文字
- 数据数值
- 坐标轴刻度
- 公式
- 图例
- 可复用模块名
- 论文中需要被审稿人精确阅读的信息

这些内容必须放在 editable overlay 中：draw.io 文本、TikZ 节点、PGFPlots 标注、Word/PPT 原生文本或形状。

## 4. 布局规划

画图指令至少包含：

- 领域和术语。
- 格式选择及理由。
- 整体布局：横向流水线、分层架构、三栏映射、对比、反馈环、时间线等。
- 模块列表：名称、重要性、相对大小、颜色、位置。
- 连线逻辑：数据流、控制流、反馈流要用不同线型或颜色区分。
- 视觉重点：核心模块更大、更醒目；辅助模块更轻。
- ASCII 草图或简短空间描述。

有参考图时先测量比例，不要凭感觉复刻：

- 画布宽高比。
- 各 zone 占比。
- 核心框宽高比。
- 主要留白比例。
- 图表/小面板在模块中的面积占比。

## 5. 信息密度和层次

好的科研图不应只是几个相同方框直连。根据任务复杂度主动增加信息层次：

- 核心模块比辅助模块更大，必要时内部展开子结构。
- 横向多 stage 优先于长竖向堆叠。
- 每个逻辑分区可使用浅色 zone 背景和 stage 标签。
- 复杂机制图可加入小型嵌入图：曲线、矩阵、热力图、柱状图、代码片段、日志片段、结果面板。
- 底部可加入 pipeline 总结条或实验结果小面板。
- 对比图要让左右两侧视觉重量接近。

最低要求不是固定元素数量，而是图的复杂度要配得上用户材料的复杂度。

## 6. 论文简洁风

当图用于毕业论文、学位论文、正式 Word 文档或黑白打印：

- 白底优先。
- 黑灰线条为主。
- 至多一种低饱和强调色。
- 避免渐变、阴影、卡片式背景、大面积色块和装饰图标。
- 图内不放大标题，标题交给图题或文档标题。
- 图内只保留短标签。
- 案例图应放真实证据：数据、代码片段、日志、IR、测试输出或结果表。

## 7. TikZ 注意事项

- 中文文本保持水平，避免旋转中文。
- 中文不要放进 `\texttt{}`。
- 编译前确认 CJK 字体可用。
- 优先用 `-|`、`|-` 等正交路径，减少手写中间点。
- 背景填充和装饰放 background 层，文字放 main 层。
- 树状分叉用连续路径画主干和横杆，再画分支箭头。
- 复杂图先用 named styles 定义节点、箭头、zone、标签。
- 生成 PDF 后必须看渲染结果，不能只看 `.tex` 代码。

## 8. draw.io 注意事项

- 适合 Office 友好编辑、技术路线图、研究框架图、展示型科研图。
- 结构要分组，zone 和 stage 命名清楚。
- 文本框不要被箭头穿过。
- 有生成图片层时，把图片作为底层或面板层，标签和箭头单独可编辑。
- 导出 SVG 用于矢量插入；含位图层时导出 PNG。
- 保留 `.drawio` 源文件。

## 9. 数据图注意事项

- 保留数据源或脚本。
- 轴标签、单位、图例和数值必须准确。
- 不用装饰性 3D 图表。
- 少量类别可直接标注，不一定需要图例。
- 数据可视化中的颜色应表达分组、强弱或对比，不做纯装饰。

## 10. 渲染后质检

交付前必须检查渲染资产：

- 文字是否可读。
- 是否有裁切、溢出、重叠。
- 箭头是否穿过文本。
- 同级元素间距是否一致。
- 左右/上下视觉重量是否平衡。
- 生成图片中是否意外出现可读文字。
- 插入 Word/PPT 后尺寸是否合适。
- 源文件是否保留，关键内容是否仍可编辑。

如果连续修补仍无法解决布局问题，回到画图指令阶段重设布局。
